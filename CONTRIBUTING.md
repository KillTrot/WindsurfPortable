# Contributing to WindsurfPortable

First off, thank you for considering contributing to WindsurfPortable! It's people like you that make the open-source community such a fantastic place to learn, inspire, and create.

We welcome all contributions, from bug reports and feature requests to code contributions and documentation improvements.

## 📝 1. The Contributor License Agreement (CLA)

To protect both you as a contributor and the long-term maintainability of the project, **all contributors must sign our Contributor License Agreement (CLA)** before a Pull Request can be merged. 

This agreement ensures that WindsurfPortable retains ownership of the merged code while allowing you to retain the right to use your own contributions for any other purpose.

**How to sign:**
You do not need to do this in advance. When you open your first Pull Request, a bot (CLA Assistant) will automatically comment on your PR with a link to read and sign the agreement using your GitHub account. It takes just a few seconds!

## 🚀 2. How to Contribute Code

If you want to submit code, please follow this standard GitHub workflow:

1. **Fork the repository:** Click the "Fork" button at the top right of the WindsurfPortable repository.
2. **Clone your fork:** `git clone https://github.com/YOUR-USERNAME/WindsurfPortable.git`
3. **Create a branch:** Create a dedicated branch for your feature or bug fix: `git checkout -b feature/your-feature-name`
4. **Make your changes:** Write your code, ensuring it aligns with the existing style and architecture.
5. **Commit your changes:** Write clear, concise commit messages.
6. **Push to your fork:** `git push origin feature/your-feature-name`
7. **Open a Pull Request:** Go to the original WindsurfPortable repository and click "New Pull Request". Compare your feature branch against our `main` branch.

## ⚖️ 3. Licensing

By contributing to WindsurfPortable, you agree that your contributions will be licensed under its **GNU General Public License v3.0 (GPLv3)**. 

This means the project remains free and open-source, and any derivative works or forks must also be shared publicly under the same terms. 

## 🐛 4. Reporting Bugs & Requesting Features

If you aren't writing code but found an issue or have a great idea:
* Check the **Issues** tab to see if it has already been reported.
* If not, open a new Issue. Please provide as much detail as possible, including steps to reproduce bugs or clear use cases for new features.
